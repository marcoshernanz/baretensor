# Project setup (modern stack)

This repo is intentionally "from scratch": zero runtime deps, native backend (C++/CUDA), and a thin Python layer.

The goal of setup is to make iteration boring:

- `uv run ...` always uses the right Python
- `ruff` formats/lints the Python layer
- `pyright` catches shape/device/dtype API mistakes early
- CMake builds the native extension against the same interpreter you import with

## Recommended repo layout

Keep Python and native code separated, and make import paths unambiguous:

```
pyproject.toml
CMakeLists.txt
.gitignore
uv.lock          # generated by uv (commit it)
milestones/
  a1.md
src/
  bt/
    __init__.py
    tensor.py
    nn/
    optim.py
native/
  include/bt/
  cpu/
  cuda/
  python/
third_party/
  nanobind/
tests/
notes/
build/            # gitignored (CMake build dir)
```

Why this organization works well:

- `src/` layout avoids accidentally importing from the repo root.
- `native/` stays build-system-focused (CMake + C++ + CUDA) and is easy to vendor/port.
- `tests/` can remain pure-stdlib (`unittest`) to keep the "zero deps" discipline honest.

## Tooling versions

"Modern but boring" baseline (good today, likely good for months):

- Python: 3.13.x recommended (3.12+ acceptable if your toolchain is picky)
- uv: latest stable
- ruff: latest stable
- pyright: latest stable
- CMake: 3.28+
- C++: C++20 (CUDA toolchains still lag C++23)

## 1) System prerequisites

macOS (CPU-only dev):

- Xcode Command Line Tools: `xcode-select --install`
- CMake + Ninja (recommended): `brew install cmake ninja`
- uv: `brew install uv` (or install via Astral's script)

Linux (CUDA box):

- A recent clang/gcc + CMake + Ninja
- CUDA toolkit 12.x for the RTX 2060 Super (SM 7.5)
- uv

## 2) Python environment with uv

From repo root:

```bash
uv python install 3.13
uv venv --python 3.13
```

Then wire dev tools (no runtime deps) via a dev dependency group in `pyproject.toml`.

Minimal shape of what you want (illustrative only):

```toml
[project]
name = "baretensor"
version = "0.0.0"
requires-python = ">=3.13"
dependencies = []

[dependency-groups]
dev = ["ruff", "pyright"]

[tool.ruff]
target-version = "py313"
line-length = 100

[tool.pyright]
pythonVersion = "3.13"
include = ["src", "tests"]
```

Install them:

```bash
uv sync --dev
```

How you run everything from now on:

- Format: `uv run ruff format src tests`
- Lint: `uv run ruff check src tests`
- Types: `uv run pyright`

Tip: until you package/install the project, use `PYTHONPATH=src` when running Python:

```bash
PYTHONPATH=src uv run python -c "import bt; print(bt)"
```

## 3) Native build (CMake + nanobind)

Key rule: build the extension against the exact Python you'll import with.

Get the interpreter path:

```bash
uv run python -c "import sys; print(sys.executable)"
```

Configure CMake using that Python executable:

```bash
cmake -S . -B build -G Ninja -DPython_EXECUTABLE=/ABS/PATH/TO/python
cmake --build build
```

Your CMake should place the built extension into the importable package dir (recommended):

- output: `src/bt/_C*.so` (macOS/Linux)
- Python imports it as `import bt._C as _C`

If you don't do this, you'll end up fighting `PYTHONPATH` and "why is import finding the wrong module?"

## 4) Test runner discipline

Keep tests dependency-free:

```bash
PYTHONPATH=src uv run python -m unittest -v
```

For early milestones, prioritize tiny tests:

- import works
- tensor creation roundtrips
- one-op forward/backward correctness via finite differences (gradcheck)

## 5) Suggested `.gitignore`

At minimum, ignore:

- `build/`
- `__pycache__/` and `*.pyc`
- `.venv/` (if you keep the venv in-repo)
- platform binaries (`*.so`, `*.dylib`, `*.pyd`) if you ever build outside `src/bt/`

## Two-machine workflow (recommended)

- MacBook: CPU-only correctness + tests + gradcheck + API iteration.
- RTX desktop: CUDA kernels + performance + long runs.

Design goal: every op has a CPU reference path, so CUDA never blocks correctness.
