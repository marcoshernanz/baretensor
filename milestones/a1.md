# A1: Build + import works

Goal: from a clean checkout, you can build the native extension and run:

Prereq: follow `SETUP.md` so `uv` and your CMake build use the same Python.

```bash
PYTHONPATH=src uv run python -c "import bt; x = bt.tensor([[1,2],[3,4]]); print(x)"
```

## What you should know

- Python import mechanics: a package (`bt/`) can import a compiled extension (`.so`/`.dylib`) as a module.
- C++ extension basics: symbol naming + module init + ABI must match the Python interpreter you import with.
- Ownership/lifetimes: Python objects must keep C++ storage alive (typically `shared_ptr` or nanobind holder types).
- Tensor v0 data model: contiguous CPU buffer + shape; everything else (strides/views/autograd) can wait.

## How to approach it

1) Pick a minimal module layout

- Recommended: a Python package `bt/` that imports a native module named something like `bt._C`.
- Keep the public surface tiny for now:
  - `bt.tensor(py_obj, dtype="f32", device="cpu", requires_grad=False)`
  - `Tensor.shape`, `Tensor.dtype`, `Tensor.device`
  - `Tensor.tolist()` or `__repr__` so you can see values

2) Implement the smallest Tensor that proves the toolchain

- Input parsing: accept nested Python lists (or any `Sequence`) and infer shape.
- Storage: contiguous `float` buffer on CPU.
- Validation: reject ragged nested lists early with a clear error.
- Keep the invariants explicit:
  - `numel == product(shape)`
  - buffer length matches `numel`

3) Build system: make "one interpreter in, one extension out"

- Use the same Python executable for:
  - configuring CMake (so it finds the right headers/libs)
  - running `import bt`
- If you see "module compiled against API version ...", you built against a different Python.

4) Add a tiny smoke test (before you write more code)

- `tests/test_a1_import.py` (stdlib `unittest`):
  - `import bt` succeeds
  - `bt.tensor([[1,2],[3,4]])` has shape `(2,2)`
  - `tolist()` roundtrips exactly (or within float tolerance)

## Common pitfalls (and how to debug)

- Import error on macOS (`dyld` / missing symbols): print `sys.executable`, rebuild, and inspect the produced `.so` with `otool -L`.
- Wrong extension location: ensure your build outputs into the package directory (or your `PYTHONPATH` includes it).
- Silent shape bugs: start with strict shape inference and fail fast on ragged inputs.

## "Done" checklist

- `PYTHONPATH=src uv run python -c "import bt"` works.
- `PYTHONPATH=src uv run python -c "import bt; print(bt.tensor([[1,2],[3,4]]))"` prints something sensible.
- A committed smoke test covers import + basic tensor creation.
