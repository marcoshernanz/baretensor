# A1: Build + import works

Goal: from a clean checkout, you can build a nanobind extension and run:

```bash
PYTHONPATH=src uv run python -c "import bt; print(bt.add(1, 2))"
```

## What you should know

- Python import mechanics: a package (`bt/`) can import a compiled extension (`.so`/`.dylib`).
- C++ extension basics: `NB_MODULE(name, m)` must match the built module name.
- ABI sanity: build against the same Python interpreter you import with.

## Docs to look for

- nanobind:
  - Getting started + module definition (`NB_MODULE`)
  - CMake integration / recommended build flags
- CMake:
  - `FindPython` / `Python::Module` and how CMake finds the active interpreter
  - macOS notes on `@rpath` and shared libraries (if you hit import-time loader errors)
- CPython (only if you need to debug import issues):
  - "Extending and Embedding the Python Interpreter" (extension module loading)

## How to approach it

1) Pick a minimal module layout

- Recommended for this repo: `src/bt/` (Python package) imports `src/bt/_C*.so` (native module).
- Keep the public surface tiny: one function `bt.add(a: int, b: int = 1) -> int`.

2) Build system: make "one interpreter in, one extension out"

- Use the same Python executable for:
  - configuring CMake (so it finds the right headers/libs)
  - running `import bt`
- If you see "module compiled against API version ...", you built against a different Python.

3) Add a tiny smoke test (before you write more code)

- `tests/test_a1_nanobind_add.py` (stdlib `unittest`):
  - `import bt` succeeds
  - `bt.add(1, 2) == 3`

## Common pitfalls (and how to debug)

- Import error on macOS (`dyld` / missing symbols): print `sys.executable`, rebuild, and inspect the produced `.so` with `otool -L`.
- Wrong extension location: ensure your build outputs into the package directory (or your `PYTHONPATH` includes it).
- Silent shape bugs: start with strict shape inference and fail fast on ragged inputs.

## "Done" checklist

- `PYTHONPATH=src uv run python -c "import bt"` works.
- `PYTHONPATH=src uv run python -c "import bt; print(bt.add(1, 2))"` prints `3`.
- A committed smoke test covers import + `add()`.

Next milestone (A2) should not require touching build plumbing; if you still find yourself fighting CMake/import issues, stop and make the build path boring first.
